# Что делать сейчас — по шагам (для начинающих)

Ниже всё разложено по шагам: как запустить игру, как проверить, что всё работает, и что делать, когда захочешь выложить приложение в ВКонтакте.

---

## Часть 1. Запустить игру у себя на компьютере

### Шаг 1. Открыть папку проекта

- Открой папку с игрой в Cursor (или в VS Code).
- Должны быть видны файлы: `package.json`, папки `app`, `components`, `lib` и т.д.

### Шаг 2. Установить зависимости (один раз)

В терминале (внизу в Cursor: Terminal → New Terminal) выполни:

```bash
npm install
```

Дождись окончания. Ошибок быть не должно.

### Шаг 3. Запустить игру

В том же терминале:

```bash
npm run dev
```

Появится что-то вроде: `Local: http://127.0.0.1:3000`.

### Шаг 4. Открыть игру в браузере

- Открой браузер (Chrome, Safari и т.д.).
- В адресной строке введи: **http://127.0.0.1:3000**
- Нажми Enter.

Должен открыться первый экран с кнопками: **Войти**, **Лидеры**, **Ставки**.

**Что можно сделать сейчас:**
- Нажать **Войти** — войдёт тестовый пользователь (без реального ВК).
- Посмотреть **Лидеры** и **Ставки**.
- Поиграть: создать ставку, выбрать соперника, играть в камень-ножницы-бумага.
- Зайти в **Профиль** (через меню внизу) и в самом низу нажать **Выйти**, чтобы вернуться на экран входа.

**Итог:** игра уже работает у тебя на компьютере. Это нужно для проверки и разработки.

## Часть 2. Что у тебя уже есть (без программирования)

- Экран входа с кнопками Войти, Лидеры, Ставки.
- Вход «как в ВК» (пока тестовый пользователь).
- Игра: ставки, арена, победа/поражение, «кто-то уснул».
- Магазин (покупки за голоса), вывод голосов (кнопка в профиле).
- Профиль с кнопкой **Выйти** внизу.

Всё это уже написано и должно работать при запуске по шагам выше.

## Часть 3. Когда захочешь выложить игру в ВКонтакте

Тогда понадобится сделать ещё несколько вещей. Их можно делать не сразу, а когда будешь готов к публикации.

### Шаг 1. Создать приложение ВКонтакте

1. Зайди на сайт ВКонтакте и открой: [vk.com/apps?act=manage](https://vk.com/apps?act=manage).
2. Нажми **Создать приложение**.
3. Выбери тип: **Мини-приложение** (или **Игры**, если такой пункт есть).
4. Заполни:
   - **Название** — например: «Камень Ножницы Бумага».
   - **Описание** — коротко, что это за игра.
5. Пока можно не указывать адрес приложения — его добавишь, когда будет готовый адрес в интернете (см. ниже).
6. Сохрани и **запиши ID приложения** (число) — он понадобится потом для платежей.

### Шаг 2. Выложить игру в интернет (хостинг)

Сейчас игра крутится только у тебя на компе. Чтобы ВК мог её открыть, нужен адрес в интернете, например: `https://твой-сайт.ru`.

**Варианты:**

#### Вариант А. Свой сервер (VPS, выделенный сервер, домашний ПК с белым IP)

Если у тебя уже есть сервер (Linux) с доступом по SSH, можно поднять игру там.

**Что должно быть на сервере:**
- Установленный **Node.js** (версия 18 или новее). Проверка: в терминале на сервере выполни `node -v`.
- Для ВК обязателен **HTTPS** — значит нужен домен, привязанный к серверу, и SSL-сертификат (например, бесплатный Let's Encrypt через nginx или certbot).

**Шаги по порядку:**

1. **Подключись к серверу по SSH** (с своего компьютера):
   ```bash
   ssh пользователь@адрес_твоего_сервера
   ```

2. **Скопируй проект на сервер.** Варианты:
   - Через Git: на сервере выполни `git clone ссылка_на_твой_репозиторий`, если проект в GitHub/GitLab.
   - Или залей папку проекта через FTP/SFTP (FileZilla, WinSCP и т.п.).

3. **На сервере зайди в папку проекта** и выполни:
   ```bash
   npm install
   npm run build
   ```
   Дождись окончания без ошибок.

4. **Запусти игру в режиме «продакшен»:**
   ```bash
   npm run start
   ```
   Игра будет слушать порт 3000. Чтобы она работала после закрытия терминала, лучше запускать через менеджер процессов, например **pm2**:
   ```bash
   npm install -g pm2
   pm2 start npm --name "rps-game" -- start
   pm2 save
   pm2 startup
   ```

5. **Настрой веб-сервер (nginx) и HTTPS:**
   - Настрой nginx как обратный прокси на `http://127.0.0.1:3000`.
   - Для домена включи SSL (например, certbot для Let's Encrypt).
   - Тогда снаружи игра будет открываться по `https://твой-домен.ru`.

6. **Адрес приложения** — это и будет `https://твой-домен.ru`. Его нужно прописать в настройках приложения ВК (Шаг 3 ниже).

**Кратко:** на своём сервере ты делаешь то же самое, что и локально (`npm install`, `npm run build`, `npm run start`), но сервер всегда включён и доступен в интернете по домену с HTTPS.

---

#### Вариант А1. Сервер на Reg.ru (VPS или «Облако»)

Если ты хочешь всё делать на **Reg.ru** (и домен, и сервер):

1. **Закажи облачный сервер (VPS) с Node.js**
   - Зайди на [reg.ru](https://www.reg.ru) → раздел «Облачные серверы» / «VPS».
   - У Reg.ru есть вариант с **предустановленным Node.js** — так проще: [reg.ru/cloud/nodejs](https://www.reg.ru/cloud/nodejs).
   - Либо закажи обычный VPS на Ubuntu и установи Node.js сам (в личном кабинете и в базе знаний Reg.ru есть инструкции).

2. **Домен**
   - Домен можно купить на Reg.ru и привязать к этому же серверу (в настройках домена указать IP сервера или настроить через панель Reg.ru).

3. **Подключись к серверу по SSH**
   - В личном кабинете Reg.ru посмотри IP сервера, логин и пароль (или ключ SSH).
   - С компьютера выполни: `ssh логин@IP_адрес_сервера`.

4. **Залей проект на сервер**
   - Через **Git**: если проект в GitHub/GitLab, на сервере: `git clone ссылка_на_репозиторий`, затем `cd имя_папки`.
   - Или через **SFTP** (FileZilla, WinSCP): подключись к серверу по SFTP и загрузи папку с игрой (где лежат `package.json`, `app`, `components` и т.д.).

5. **На сервере в папке проекта выполни:**
   ```bash
   npm install
   npm run build
   npm run start
   ```
   Проверь, что по адресу `http://IP_твоего_сервера:3000` игра открывается.

6. **Чтобы игра не останавливалась после выхода из SSH**, запусти через **pm2**:
   ```bash
   npm install -g pm2
   pm2 start npm --name "rps-game" -- start
   pm2 save
   pm2 startup
   ```

7. **HTTPS и домен (для ВК обязателен https)**
   - Установи **nginx** и настрой его: он должен принимать запросы по твоему домену и перенаправлять их на `http://127.0.0.1:3000`.
   - Выпусти **SSL-сертификат** (бесплатный Let's Encrypt через certbot или через панель Reg.ru, если там есть «SSL»).
   - В настройках домена на Reg.ru укажи **A-запись** на IP твоего сервера.

8. **Адрес приложения для ВК** — это будет `https://твой-домен.ru` (или тот адрес, который открывает твою игру по HTTPS).

Если используешь **общий виртуальный хостинг** Reg.ru (не VPS) — там обычно нет поддержки Node.js, только PHP. Для этой игры нужен именно **VPS или облачный сервер** с Node.js.

**Если не получается подключиться к серверу Reg.ru (FileZilla и т.п.)**

Частая причина — подключаются как к обычному FTP, а на VPS используется **SFTP** (по SSH). Сделай по шагам:

1. **Убедись, что подключаешься по SFTP, а не по FTP**
   - В FileZilla: **Файл → Менеджер сайтов → Новый сайт**.
   - В поле **Протокол** выбери **SFTP — SSH File Transfer Protocol** (не FTP!).
   - **Хост** — IP-адрес сервера (только цифры, например `123.45.67.89`).
   - **Порт** — **22** (оставь пустым или явно укажи 22).
   - **Тип входа** — «Обычный» или «Запросить пароль».
   - **Пользователь** — чаще всего **root** (или логин из письма/панели Reg.ru).
   - **Пароль** — пароль от сервера (из письма Reg.ru или из личного кабинета).
   - Нажми «Подключить».

2. **Где взять IP, логин и пароль (если в письме нет)**
   - Зайди на [reg.ru](https://www.reg.ru) и **войди в личный кабинет** (логин и пароль от аккаунта Reg.ru).
   - В верхнем меню или в боковой колонке найди раздел **«Серверы»**, **«VPS»** или **«Облако»** / **«Облачные серверы»**.
   - Открой его — появится список твоих серверов. Нажми на **название сервера** или **«Управление»**.
   - На странице сервера ищи:
     - **IP-адрес** — строка из цифр и точек (например `185.10.20.30`). Может быть в блоке «Сеть», «Доступ» или сразу в шапке карточки.
     - **Логин** — часто указан как «Пользователь» или «User», обычно **root**.
     - **Пароль** — кнопка **«Показать пароль»** или **«Сбросить пароль»** (сброс пришлёт новый пароль на почту или покажет в кабинете).
   - Если раздела «Серверы» не видно — внизу страницы личного кабинета часто есть **«Все услуги»** или **«Мои услуги»**: там будут и домены, и серверы. Выбери свой VPS/облачный сервер и открой его — IP будет на странице этого сервера.
   - Пароль при первом создании сервера иногда приходит **отдельным письмом** от Reg.ru (проверь папку «Спам» и поиск по «reg.ru», «пароль», «сервер»).

3. **Сначала проверь подключение по SSH** (так проще понять, что сервер вообще доступен)
   - На **Windows**: открой **PowerShell** или установи [PuTTY](https://www.putty.org/) / клиент из Microsoft Store «OpenSSH».
   - На **Mac**: открой **Терминал**.
   - Выполни (подставь свой IP и логин):
   ```bash
   ssh root@IP_АДРЕС_СЕРВЕРА
   ```
   - Введи пароль, когда спросит. Если зашёл — видишь приглашение вроде `root@server:~#`. Значит, сервер доступен, и в FileZilla нужно просто верно указать SFTP, порт 22 и эти же логин/пароль.
   - Если SSH пишет «Connection refused» или «timed out» — см. пункт 4.

4. **Если и SSH не подключается**
   - **Брандмауэр**: в панели Reg.ru у сервера посмотри раздел «Сеть» / «Фаервол» — порт **22** (SSH) должен быть открыт для входящих.
   - **Сервер ещё создаётся**: после заказа VPS иногда нужно 5–10 минут, пока он запустится. Статус смотри в личном кабинете.
   - **Сброс пароля**: в карточке сервера в Reg.ru бывает кнопка «Сбросить пароль» — получи новый пароль и попробуй снова.

5. **Альтернатива FileZilla — WinSCP (Windows)**
   - Скачай [WinSCP](https://winscp.net/). При создании сессии выбери протокол **SFTP**, хост = IP, порт 22, логин и пароль. Подключение часто получается с первого раза.

6. **Написать в поддержку Reg.ru**
   - В личном кабинете есть «Помощь» / «Поддержка». Напиши: «Купил VPS, не могу подключиться по SFTP (порт 22) и по SSH. Подскажите, какой логин и пароль использовать и открыт ли порт 22». Укажи ID сервера или домен.

#### Вариант Б. Бесплатный облачный хостинг (Vercel, Netlify)

- Зарегистрируйся на сайте (например, [vercel.com](https://vercel.com)).
- Подключи свой проект (папку с игрой или репозиторий) — там есть «Import project» и подсказки.
- Сервис сам соберёт проект и даст ссылку вида `https://твой-проект.vercel.app`.
- Эту ссылку потом пропиши в настройках приложения ВК как **адрес приложения**.

**Важно:** для ВК нужен именно **https**. И на своём сервере, и на Vercel/Netlify его нужно настроить (на своём — через nginx + Let's Encrypt, в облаке — обычно уже есть).

### Шаг 3. Указать адрес приложения в ВК

1. Снова зайди в [vk.com/apps?act=manage](https://vk.com/apps?act=manage).
2. Открой своё приложение.
3. В настройках найди поле **Адрес приложения** (или **URL приложения**).
4. Вставь туда ссылку с твоего хостинга, например: `https://твой-проект.vercel.app`.
5. Сохрани.

После этого приложение можно открыть из ВК по этой ссылке (ВК откроет его в своём окне).

### Шаг 4. Реальные платежи и вывод денег (если нужны)

Сейчас пополнение и вывод в игре **симулируются**: голоса прибавляются/списываются только у тебя в браузере, без реальных денег ВК.

Чтобы **по-настоящему** принимать оплату голосами ВК и выводить деньги:

- Нужен **сервер (бэкенд)** с твоим кодом, который:
  - для пополнения: выдаёт подпись для платежа ВК и после успешной оплаты начисляет голоса пользователю;
  - для вывода: принимает заявки на вывод, проверяет лимиты и только потом списывает голоса.
- Это уже программирование (Node.js, Python и т.п.) или помощь того, кто умеет делать бэкенды.

Подробности для разработчика лежат в папке **docs**:  
**VK_INTEGRATION.md** и **PUBLICATION_VK.md**.

## Если что-то пошло не так

- **Не запускается `npm run dev`** — проверь, что ты в терминале находишься в папке проекта (там, где лежит `package.json`). Команда `cd путь/к/папке/проекта` переводит в нужную папку.
- **Страница не открывается** — убедись, что в браузере введён именно адрес `http://127.0.0.1:3000` (без лишних букв и с правильными цифрами).
- **Ошибка при `npm install`** — напиши текст ошибки; по нему можно подсказать следующий шаг.

Можешь ориентироваться на эту инструкцию и выполнять только те шаги, которые тебе нужны сейчас (например, только Часть 1 для начала).
