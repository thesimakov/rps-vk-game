# Хранимые процедуры ВК: что прописать

По [документации ВК](https://dev.vk.com/ru/mini-apps/settings/development/stored-procedures).

---

## 1. Название процедуры

**Где:** при создании процедуры в разделе **Разработка → Хранимые процедуры** (кнопка «Добавить процедуру»).

**Что прописать:**
- Только **латинские буквы** (и при необходимости цифры/подчёркивание).
- Длина: **от 1 до 100 символов**.
- Примеры: `getUserBalance`, `getLeaderboard`, `saveScore`.

**Важно:** при вызове в коде ВК всегда добавляет префикс `execute.` — его менять нельзя.  
То есть процедура с названием `getUserBalance` вызывается как метод **execute.getUserBalance**.

В коде проекта передаётся только имя без префикса:
```ts
callStoredProcedure("getUserBalance", { user_id: 123 })
// → вызывается execute.getUserBalance
```

## 2. Версия процедуры

**Где:** в карточке процедуры — выпадающий список **«Версия процедуры»** (появляется после первого сохранения).

**Что прописать:**
- У одной процедуры может быть **несколько версий** (разный код).
- По умолчанию создаётся одна версия.
- Чтобы добавить ещё: выберите исходную версию → **«Добавить версию»** → введите новый код и сохраните.
- Название/идентификатор версии задаётся платформой (например, «Версия 1», «Версия 2»).

**Когда нужна:** если вы меняете код процедуры, а старые клиенты ещё вызывают старый контракт — оставьте старую версию и создайте новую; в новом коде приложения вызывайте новую версию через параметр `funcVersion`.

В коде проекта версия передаётся опционально:
```ts
callStoredProcedure("getUserBalance", { user_id: 123 }, { funcVersion: "Версия 2" })
```
Если не передавать `funcVersion`, выполнится текущая/дефолтная версия процедуры.

## 3. Версия API

**Где:**  
- в настройках процедуры — поле **«Версия API»** (при создании/редактировании процедуры);  
- при вызове — параметр **`v`** в запросе.

**Что прописать:**
- Номер версии API ВКонтакте, которой соответствуют методы и форматы данных, используемые **внутри** процедуры (например `5.134`, `5.199`).
- Должна совпадать с реальной версией методов, которые вызываются в VKScript (см. [документацию API](https://dev.vk.com/reference)).

В проекте по умолчанию используется константа **VK_API_VERSION** (`lib/vk-stored-procedures.ts`), сейчас это **5.134**. При вызове её можно переопределить:
```ts
callStoredProcedure("getUserBalance", { user_id: 123 }, { apiVersion: "5.199" })
```

**Итог:** в настройках процедуры в панели ВК укажите ту же версию API (например 5.134), что и в коде при вызове (`v` / `apiVersion`).

## Краткая таблица

| Поле                 | Где задаётся                    | Что прописать                          | В коде проекта                          |
|----------------------|----------------------------------|----------------------------------------|----------------------------------------|
| **Название процедуры** | Название при создании процедуры | Латиница, 1–100 символов (напр. `getUserBalance`) | `callStoredProcedure("getUserBalance", params)` — без префикса `execute.` |
| **Версия процедуры**  | Список «Версия процедуры» в карточке процедуры | Одна или несколько версий кода; идентификатор версии (напр. «Версия 1») | Опционально: `options.funcVersion: "Версия 2"` |
| **Версия API**        | Поле «Версия API» в настройках процедуры | Версия API ВК (напр. `5.134`)          | По умолчанию `VK_API_VERSION` (5.134), либо `options.apiVersion` |
