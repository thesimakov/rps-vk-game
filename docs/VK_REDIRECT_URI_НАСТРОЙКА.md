# Куда вписать Redirect URI в настройках приложения ВКонтакте

Ошибка **«Security Error»** при входе через ВК возникает, когда адрес возврата (Redirect URI) не указан в приложении или указан неверно. Ниже — где именно его задать.

---

## Вариант 1: Управление приложениями ВК (vk.com)

Если вы создавали приложение здесь: **[vk.com/apps?act=manage](https://vk.com/apps?act=manage)** (раздел «Мои приложения»):

1. Откройте **[vk.com/apps?act=manage](https://vk.com/apps?act=manage)** и выберите своё приложение (или создайте новое).
2. Перейдите в **«Настройки»** приложения (вкладка или пункт меню слева).
3. Найдите блок **«Платформы»** (или «Настройки платформы»).
4. Выберите платформу **«Веб»** / **«Сайт»** / **«Веб-сайт»**.  
   Если такой платформы нет — добавьте её (кнопка «Добавить платформу» или «Подключить сайт»).
5. В настройках веб-платформы заполните:
   - **«Адрес сайта»** или **«Redirect URI»** или **«Доверенный redirect URI»** — подставьте **полный** адрес страницы, на которую ВК будет возвращать пользователя после входа, например:
     - для локальной разработки: `http://localhost:3000/vk-callback`
     - для продакшена: `https://ваш-домен.ru/vk-callback`
   - **«Базовый домен»** — домен без протокола и пути, например: `localhost:3000` или `ваш-домен.ru`.
6. Сохраните настройки.

Точное название полей может отличаться (например, «Доверенный redirect URL», «Authorized redirect URI»). Ищите поле, в котором указывается **полный URL возврата после авторизации**.

---

## Вариант 2: VK ID (id.vk.com)

Если приложение создано в кабинете **VK ID**: **[id.vk.com](https://id.vk.com)** (или id.vk.ru):

1. Зайдите в кабинет VK ID и откройте **«Мои приложения»**.
2. Выберите нужное приложение.
3. В настройках приложения найдите блок **«Доверенный redirect URL»** (или «Authorized redirect URI», «Redirect URI»).
4. Добавьте туда **один точный** адрес, например:
   - `http://localhost:3000/vk-callback`
   - или `https://ваш-домен.ru/vk-callback`
5. Сохраните изменения.

---

## Что вписать в поле Redirect URI

Вписать нужно **адрес главной страницы приложения** — на него ВК вернёт пользователя после входа (токен придёт в hash в URL). Для этого проекта:

- **Локально:** `http://localhost:3000/`  
  (если открываете по `http://127.0.0.1:3000`, укажите `http://127.0.0.1:3000/`.)
- **На сервере:** `https://ваш-домен.ru/`  
  (замените на свой домен.)

Без параметров в конце. Если в .env задан `NEXT_PUBLIC_VK_REDIRECT_URI`, он должен **полностью совпадать** с тем, что указано в настройках ВК.

---

## Если такого раздела нет

- Убедитесь, что открыто приложение типа **«Сайт»** / **«Веб»** (для мини-приложений и игр иногда нужно отдельно добавить платформу «Сайт»).
- Обновите страницу настроек или зайдите в приложение заново с [vk.com/apps?act=manage](https://vk.com/apps?act=manage).
- В старом интерфейсе ищите: **Настройки → Платформы → Веб-сайт** и поля **«Адрес сайта»** / **«Базовый домен»** — Redirect URI часто задаётся как «Адрес сайта» (полный URL возврата).

После сохранения правильного Redirect URI ошибка «Security Error» должна пропасть при следующей попытке входа.
