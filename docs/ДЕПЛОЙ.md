# Размещение игры на сервере и во ВКонтакте

Краткая пошаговая инструкция: сборка → хостинг → настройка приложения ВК.

---

## Шаг 0. Сборка проекта

В папке проекта выполни:

```bash
npm install
npm run build
```

После успешной сборки появится папка **`out/`** со статическими файлами (HTML, JS, CSS). Её можно выложить на любой хостинг с HTTPS.

---

## Вариант 1. Размещение на Vercel (проще всего)

1. Зарегистрируйся на [vercel.com](https://vercel.com) (можно через GitHub).
2. Нажми **Add New** → **Project** и импортируй репозиторий с игрой (или загрузи папку через Vercel CLI).
3. Параметры оставь по умолчанию: **Framework Preset: Next.js**, **Build Command: `npm run build`**, **Output Directory: `out`** (для статического экспорта).
4. Нажми **Deploy**. Через минуту получишь ссылку вида `https://твой-проект.vercel.app`.
5. Эту ссылку укажи в настройках приложения ВК (см. раздел «Настройка приложения ВК» ниже).

**Через CLI (альтернатива):**

```bash
npm i -g vercel
vercel login
vercel
```

Следуй подсказкам; в конце будет выдан URL приложения.

## Вариант 2. Размещение на своём сервере (VPS)

Нужны: сервер с Linux, Node.js 18+, домен и HTTPS (для ВК обязателен).

### 2.1 Подготовка на сервере

```bash
# Установка Node.js (Ubuntu/Debian)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### 2.2 Загрузка проекта

- Через **Git**: `git clone <url-репозитория>`, затем `cd rps-vk`.
- Или залей файлы по **SFTP** (FileZilla, WinSCP) в папку, например `/var/www/rps-vk`.

### 2.3 Сборка и раздача статики

```bash
cd /var/www/rps-vk   # или твой путь
npm install
npm run build
```

Папка **`out/`** — это готовый статический сайт. Настрой веб-сервер (nginx/Apache), чтобы он раздавал файлы из `out/` по твоему домену.

### 2.4 Пример nginx для раздачи `out/`

```nginx
server {
    listen 443 ssl http2;
    server_name твой-домен.ru;
    root /var/www/rps-vk/out;
    index index.html;
    location / {
        try_files $uri $uri/ $uri.html /index.html;
    }
    ssl_certificate     /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;
}
```

SSL можно получить бесплатно через [Let's Encrypt](https://letsencrypt.org/) (certbot). После перезапуска nginx приложение будет доступно по `https://твой-домен.ru`.

## Вариант 3. VK Hosting (хостинг ВКонтакте)

Если хочешь разместить приложение на хостинге ВК:

1. Собери проект: `npm run build` (папка **`out/`**).
2. Скопируй конфиг:  
   `cp vk-hosting-config.example.json vk-hosting-config.json`  
   Открой `vk-hosting-config.json` и подставь **ID приложения** ВК в поле `app_id`.
3. Используй [@vkontakte/vk-miniapps-deploy](https://www.npmjs.com/package/@vkontakte/vk-miniapps-deploy) или загрузи содержимое папки `out/` вручную в разделе «Файлы» приложения ВК (если такой функционал доступен в [Управлении приложениями](https://vk.com/apps?act=manage)).

Адрес приложения тогда будет выдан ВК (например, `https://vk.com/app123456789` или URL вида `https://vk-hosted-app...` — смотри в настройках приложения).

## Настройка приложения ВКонтакте

1. Открой [Управление приложениями ВК](https://vk.com/apps?act=manage).
2. **Создать приложение** → тип **Мини-приложение** (или **Игры**).
3. Заполни:
   - **Название** — например: «Камень Ножницы Бумага».
   - **Краткое описание** — для каталога.
   - **Адрес приложения** — URL, где открывается игра:
     - Vercel: `https://твой-проект.vercel.app`
     - Свой сервер: `https://твой-домен.ru`
     - VK Hosting: укажи адрес, выданный ВК.
   - **Базовый домен** — домен без `https://` (например, `твой-проект.vercel.app` или `твой-домен.ru`).
4. Загрузи **иконку** приложения (рекомендуется 256×256).
5. Сохрани и запиши **ID приложения** — понадобится для платежей и, при использовании VK Hosting, для `vk-hosting-config.json`.
6. В настройках приложения добавь **политику конфиденциальности** (ссылка на страницу с правилами) — это нужно для модерации.
7. Когда всё готово — **Отправить на модерацию**. После одобрения приложение появится в каталоге и по ссылке.

## Проверка перед модерацией

- [ ] Игра открывается по ссылке из настроек ВК (в мобильном приложении ВК и в браузере).
- [ ] Кнопка «Войти» работает (VKWebAppGetUserInfo).
- [ ] Нет жёстко прописанных секретов в коде.
- [ ] В настройках приложения ВК указана ссылка на политику конфиденциальности.

## Платежи и вывод (продакшен)

Сейчас пополнение и вывод **симулируются** в интерфейсе. Для реальных платежей и вывода голосов нужен **бэкенд** (сервер), который:

- для пополнения: выдаёт подпись запроса (`app_id`, `hash`) и после подтверждения оплаты ВК начисляет голоса;
- для вывода: принимает заявки, проверяет лимиты и баланс, затем списывает голоса.

Подробности: **VK_INTEGRATION.md** и **PUBLICATION_VK.md**.
