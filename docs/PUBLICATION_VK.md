# Публикация мини-приложения в ВКонтакте

Пошаговая подготовка и публикация приложения «Камень · Ножницы · Бумага» в VK Mini Apps.

---

## 1. Подготовка проекта

- **Сборка:** `npm run build` — выход в `out/` (при `output: 'export'`) или `.next` (при `next start`).
- **Домен:** приложение должно открываться по HTTPS. Домен нужно будет указать в настройках приложения ВК.
- **Переменные:** для продакшена подготовить бэкенд (API для подписи платежей и приёма заявок на вывод). См. **VK_INTEGRATION.md**.

---

## 2. Создание приложения ВК

1. Перейти в [Управление приложениями ВК](https://vk.com/apps?act=manage).
2. Создать приложение → **Мини-приложение** (или **Игры**).
3. Заполнить:
   - **Название** — например: «Камень Ножницы Бумага».
   - **Краткое описание** — для каталога.
   - **Адрес приложения** — URL, по которому открывается приложение (ваш хостинг или VK Hosting).
   - **Базовый домен** — домен без протокола (например, `myapp.example.com`).
4. Загрузить **иконку** приложения (например, 256×256).
5. Сохранить и записать **ID приложения** — понадобится для платежей и деплоя.

---

## 3. Хостинг

### Вариант A: свой сервер / Vercel / Netlify

- Собрать: `npm run build`.
- Для Next.js: либо экспорт статики (`output: 'export'` в `next.config.mjs`), либо запускать `next start` на сервере.
- Настроить HTTPS и указать в настройках ВК **Адрес приложения** (например, `https://myapp.example.com`).

### Вариант B: VK Hosting

- Использовать [@vkontakte/vk-miniapps-deploy](https://www.npmjs.com/package/@vkontakte/vk-miniapps-deploy) или ручную загрузку в хостинг ВК.
- В корне проекта можно добавить конфиг для деплоя (если используете их инструмент):

**Пример конфига** — в корне проекта есть `vk-hosting-config.example.json`. Скопируйте в `vk-hosting-config.json` и подставьте свой `app_id`:

```json
{
  "static_path": "out",
  "app_id": "ВАШ_APP_ID",
  "endpoints": {
    "mobile": "index.html",
    "mvk": "index.html",
    "web": "index.html"
  }
}
```

- Для Next.js со статическим экспортом папка сборки — `out`; точка входа — `index.html`.
- `app_id` подставить из настроек приложения ВК.

## 4. Настройки приложения в ВК

- **Права:** в настройках приложения включить права, необходимые для платежей (если используете VK Pay/голоса).
- **Категория:** указать подходящую (игры, развлечения и т.п.).
- **Политика конфиденциальности:** добавить ссылку на страницу с правилами обработки данных (обязательно для публикации).
- **Правила:** убедиться, что приложение соответствует [правилам размещения Mini Apps](https://dev.vk.com/mini-apps-rules).

---

## 5. Бэкенд для продакшена

Перед полноценной публикацией с оплатой и выводом нужен бэкенд:

1. **Платежи (пополнение голосов):**
   - Эндпоинт, который по `amount` возвращает `app_id` и `hash` (подпись по правилам ВК).
   - После успешной оплаты — callback/webhook от ВК или проверка через API ВК и начисление голосов в БД.
   - Клиент не должен начислять баланс без подтверждения с сервера.

2. **Вывод:**
   - Эндпоинт приёма заявки (user_id, amount). Проверка лимитов (мин 10, макс 10 000/день) и баланса на бэкенде.
   - После принятия заявки — ответ успех; клиент тогда списывает баланс и вызывает `recordWithdraw(amount)`.
   - Фактический перевод пользователю — через API ВК или вручную по заявкам.

Подробности: **VK_INTEGRATION.md**.

---

## 6. Проверка перед отправкой на модерацию

- [ ] Приложение открывается по ссылке из настроек ВК (в мобильном приложении ВК и в браузере).
- [ ] Вход по кнопке «Войти» (VKWebAppGetUserInfo) работает.
- [ ] Магазин и пополнение: вне ВК отображается подсказка «Откройте в ВКонтакте»; в ВК форма оплаты открывается (в проде — с app_id и hash с бэкенда).
- [ ] Вывод: отображаются лимиты и подсказка про ВК; в проде заявка уходит на бэкенд, баланс списывается только после успешного ответа.
- [ ] Нет жёстко прописанных секретов и ключей в клиентском коде.
- [ ] Есть ссылка на политику конфиденциальности в настройках приложения ВК.

---

## 7. Отправка на модерацию

В карточке приложения ВК нажать «Отправить на модерацию» и заполнить форму. После одобрения приложение станет доступно в каталоге и по прямой ссылке.
